C与C++面经汇总

C和C++的区别：

  • C是面向过程的语言，而C++是面向对象的语言，因此C++语言中有类和对象以及继承和多态等OOP语言的内容，此外C++支持模板，运算符重载，以及C++标准模板库STL。所谓C++的四大特性就是指：抽象，封装，继承，多态。这就是与C语言最大的区别。

  • C与C++在动态内存开辟上也有区别，C是通过malloc与free进行堆内存的开辟与释放；而C++是通过new与delete进行堆内存的开辟与释放。

  • C与C++在强制类型转换上也不一样，C的强制类型转换是直接使用()小括号里面写入类型来进行的；而C++提供了四种强制转换类型，分别是：const_cast，static_cast，reinterpret_cast和dynamic_cast。

  • C++还支持内联函数，可以节省简单函数的开销。

  • C与C++中对const的处理方式也不一样

  • C++不仅支持指针，还支持引用变量。

  • C与C++对函数的处理方式不同，C代码生成函数符号时，是以函数名决定的；而C++代码生成函数符号时，是以函数名+参数类型决定的，这也是C++支持函数重载的原因。

malloc/free与new/delete的区别：

  • malloc/free是C/C++标准库的函数，new/delete是C++操作符。

  • malloc/free只是动态分配内存空间/释放空间；而new/delete除了分配空间还会调用构造函数与析构函数来进行成员初始化与成员释放。

  • malloc/free需要手动计算类型大小且返回值为void*，new/delete可自己计算类型的大小对应类型的指针。

  • new/delete的底层同样调用了malloc/free。

  • new后直接跟类型，而malloc后跟字节数。

  • malloc/free申请空间后判断是否成功是与’0’比较，new/delete则是通过抛出异常来判断是否成功。如：

C++的四种类型转换：

用法：xxx_cast 变量名;

  • const_cast：去掉常量属性的类型转换
  • static_cast：提供编译器认为安全的类型转换
  • reinterpret_cast：类似于C风格的强制类型转换
  • dynamic_cast：主要用于继承结构中，支持RTTI类型识别的上下转换，常用于判断指针指向的对象是否是特定派生类类型的对象

C++中的内联函数与普通函数的区别：

定义：内联函数指在函数定义时在返回值前添加关键字inline的函数。

内联函数的特性：

  • 在编译过程中，在函数的调用点直接将函数的代码进行展开处理；

  • 内联函数只是对编译器的建议，是否处理为内联函数由编译器决定；例如递归函数与函数内部实现很复杂的函数，因为将函数处理为内联函数是为了节省调用函数时的开销，对于大量使用的简单函数处理为内联函数会大大提高系统的效率，而对于递归函数则无法直接做展开处理，或者内部实现很复杂的函数做展开处理可能会造成程序运行的错误，所以是否处理为内联函数由编译器决定。

  • 内联函数在汇编阶段不会在符号表中生成相应的符号；

  • 在debug版本上，内联函数是不起作用的；只有在release版本才会起作用；

调用函数的开销：正常函数的调用需要把形参入栈，将下一条指令地址入栈，开辟新的栈帧空间，函数处理完成后回退栈帧空间，将esp与ebp重新指向调用函数的栈顶和栈底，将下一条指令地址出栈存入PC寄存器中，释放掉调用函数栈帧中的形参变量所占用的空间，这些统称为调用函数的开销。

C与C++中对const的处理有什么不同：

在C语言中，const修饰的变量，可以不用初始化，但是为了后续使用方便，建议初始化；并且const所修饰的量不叫作常量，而叫做常变量，本质上仍是变量。即const只是在语法上保证a不可以作为左值被修改但通过指针解引用操作可以改变a所占内存中的值，同样也可以通过插入汇编指令修改a所占内存的值。

C++从源文件到可执行文件的过程：

从源文件到可执行文件，需要经过预处理，编译，汇编，链接四个过程。

1.预处理

命令：gcc -E main.c -o main.i

预处理的主要作用：

  • 删除源文件中的所有注释；

  • 删除所有的#define，展开所有的宏定义；

  • 处理所有的条件预编译指令，如“#if”、“#endif”、“#ifdef”、“#elif”和“#else”；

  • 处理“#include”预编译指令，将文件内容替换到它的位置，这个过程是递归进行的，文件中包含其他文件；

  • 添加行号和文件表示，以便于编译时编译器产生调试用的行号信息及用于编译时产生编译错误或警告时能欧够显示行号；

注意：需要保留所有的#pragma指令，因为在编译链接阶段需要使用到它，例如：

  • #pragma once 防止头文件重复引用

  • #pragma message 自定义编译信息输出到终端(一般和#if配合使用，用在控制版本号)

  • #pragma pack 指定内存对齐

2.编译

命令：gcc -S main.c -o main.s

作用：进行一系列词法分析、语法分析、语义分析及优化后，生成相应的汇编代码文件。

3.汇编

命令：gcc -c main.c

将汇编代码转变成机器可以执行的指令(机器码文件)。根据汇编指令和机器指令的对照表一一翻译过来，汇编过程有汇编器as完成。经汇编之后，会产生可重定位的二进制目标文件，Linux下为.o文件，windows下为.obj文件。

.o文件实际上是由很多段组成的，而这些段在最终的链接阶段经过合并即可形成可执行文件。

可以通过readlf -h main.o来查看该目标文件的文件头信息，可以发现，该文件类型为REL(可重定位文件)；入口点地址为0x0空地址，因为在汇编阶段并没有分配虚拟地址，同样的，符号在该阶段也没有虚拟地址，需要在链接阶段进行分配，这也是.o文件无法运行的原因之一。

可以通过objdump -t main.o来查看经过汇编生成的.o文件中的符号表(存储在.symtab段中)，可以发现，main()函数产生对应的指令存放在.text段上；main()函数外定义的data存放在.data段，因为它是全局变量且初始化不为0；而在main()函数中只进行引用的gdata与sum()也产生了符号，但它们的标识为”UND”，表示未定义，表明在当前文件中用到了这些符号，但是在当前文件中没有对它们的定义，需要在链接时处理标识为”UND”的符号；并且文件中还描述了符号的全局与局部属性，l表示local局部属性，g表示global全局属性，属性为l的符号在链接时无法被其他文件看到，全局变量与函数名所产生的符号属性为g，而静态全局变量与局部变量所产生的符号属性都为l，所以可以在不同文件中定义变量名相同的静态全局变量，但不可以是普通全局变量。

4.链接

命令：ld -e main *.o

每个源代码模块独立地编译，然后按照要求将他们组装起来，这个组装过程就是链接，并生成a.out可执行文件。

可执行目标文件相对于可重定位的目标文件的区别在于，可执行文件多了一个program headers段，内部有两个load，用于告诉系统在运行该程序时，将哪些内容加载到内存当中，可以发现只加载了代码段与数据段，将入口地址加载到CPU的PC寄存器中，即可正确运行该程序。

汇编的主要作用：

  • 将.o文件中的各个段进行合并，在合并符号表时，需要进行符号解析；

  • 符号解析成功后，进行符号的重定位，即给所有的符号分配虚拟地址，再给代码段的指令填充正确虚拟地址；

符号解析：意为所有对符号的引用，都需要找到该符号定义的地方。如上述main()文件符号表中的”UND”则需要在sum()文件的符号表中找到该定义；若没找到，则会出现符号未定义错误；若找到多个定义，则会出现符号重定义错误。

static关键字

1.全局静态变量

存储区域：若初始化不为0则存储在.data段，若未初始化或初始化为0则存储在.bss段，未初始化的会自动初始化为0。在整个程序运行期间一直存在。

作用域：全局静态变量在声明他的文件之外是不可见的，准确地说是从定义之处开始，到文件结尾。

2.局部静态变量

存储区域：若初始化不为0则存储在.data段，若未初始化或初始化为0则存储在.bss段，未初始化的会自动初始化为0。在整个程序运行期间一直存在。

作用域：作用域仍为局部作用域，当定义它的函数或者语句块结束的时候，作用域结束。但是当局部静
态变量离开作用域后，并没有销毁，而是仍然驻留在内存当中，只不过我们不能再对它进行访问，直到
该函数再次被调用，并且值不变。

3.静态函数

函数的定义和声明在默认情况下都是extern的，但静态函数只是在声明他的文件当中可见，不能被其他文件所用。所以静态函数不会与在其他cpp文件中的同名函数起冲突。

注意：不要在头文件中声明static的全局函数，不要在cpp文件中内声明非static的全局函数，如果你要在多个cpp文件中复用该函数，就把它的声明提到头文件里去，否则cpp内部声明需加上static修饰。

4.类的静态成员

类的静态成员在类内声明，在类外定义，并且不占用类的内存空间，而是存储在.data段。

在类中，静态成员可以实现多个对象之间的数据共享。静态成员是类的所有对象中共享的成员，而不是某个对象的成员。对多个对象来说，静态数据成员只存储一处，供所有对象共用。

5.类的静态成员函数

静态成员方法编译时不会生成this指针，不用通过对象来访问，它是直接通过类作用域调用的；所有的静态成员方法是所有对象共享的，但无法访问普通成员变量，只可以访问静态成员变量。

函数重载的实现机制

因为C++中的函数在经过汇编产生函数符号时，是依赖函数名与参数列表共同来产生的，即使函数名相同，但参数列表不同，在.o文件的符号表中生成的是不同的符号，其入口地址也不同，所以本质上是两个不同的函数；而C中对函数的处理仅仅依赖于函数名，所以即使参数不同编译器也会认为是同一个函数，从而产生重定义的错误。

基本数据类型的大小

  • short：2

  • int：4

  • long：4（linux下32位系统为4，64位系统为8）

  • long long：8

  • float：4

  • double：8

  • char：1

C和C++中结构体的区别：

C中的结构体只可以声明成员变量，且默认权限为public不可以更改，并且不可以继承；

C++中的结构体相当于类，只是默认权限为public，而类的默认权限为private，其他功能与类一致。

C++中结构体与类的区别：

在C++中，结构体是一种特殊形态的类，唯一的区别是：结构体和类有不同的默认访问权限，结构体为public，类为private。在C++中，使用结构体与类没有任何使用上的差别，但是保留结构体是为了保持与C代码的兼容性。

C++中的三大特性

三大特性分别指：封装，继承与多态

封装：就是将抽象得到的数据和行为相结合，形成一个有机的整体，也就是将数据与操作数据的源代码进行有机的结合，形成类，其中数据和函数都是类的成员，目的在于将对象的使用者和设计者分开，

继承：继承就是新类从已有类那里得到已有的特性。 类的派生指的是从已有类产生新类的过程。原有的类称为基类或父类，产生的新类称为派生类或子类，子类继承基类后，可以创建子类对象来调用基类函数，变量等。

继承方式如下表：

继承方式 基类的访问限定 派生类的访问限定 外部的访问限定
public public public Y
protected protected N
private 不可见的 N
protected public protected N
protected protected N
private 不可见的 N
private public private N
protected private N
private 不可见的 N

多态：多态分为静态多态与动态多态

静态多态：也称为编译期间的多态，编译器在编译期间完成的，编译器根据函数实参的类型(可能会进行隐式类型转换)，可推断出要调用那个函数，如果有对应的函数就调用该函数，否则出现编译错误。
静态多态有两种实现方式：

  • 函数重载：包括普通函数的重载和成员函数的重载
  • 函数模板的使用

动态多态（动态绑定）：即运行时的多态，在程序执行期间(非编译期)判断所引用对象的实际类型，根据虚函数表中的RTTI来进行判断，根据其实际类型调用相应的方法。

在类的构造函数中调用虚函数，也是静态绑定；对象本身访问访问自己的虚函数也不会发生动态绑定，而基类对象通过指针或者引用变量访问派生类的虚函数才会发生动态绑定。

虚函数与纯虚函数

虚函数：

如果给一个类中的成员方法的返回值前加关键字virtual，则这个函数称为”虚函数”。

如果一个类中定义了虚函数，则：

  • 在编译阶段，编译器会给每个类添加一个唯一的vftable称为”虚函数表”，虚函数表中主要存储的内容是RTTI指针和虚函数地址。并且在运行时，每一张虚函数表都会被加载到内存中的.rodata区。
  • 在运行阶段，即创建一个对象的时候，该对象内存的前四个字节会存储一个vfptr称为”虚函数表指针”，并且在程序运行时，编译器会把虚函数表中首个虚函数的地址赋值给虚函数表指针，所以可以通过虚函数指针来访问类中的虚函数。因此有虚函数的类创建的对象的内存会比普通类对象大四个字节，用来存储虚函数表指针。

需要注意的是：

  • 一个类定义的N个对象，它们的虚函数表指针指向的都是同一个虚函数表；
  • 无论类中有多少个虚函数，都不影响对象内存的大小，影响的是虚函数表的大小；

纯虚函数：

形式为virtual 函数原型 = 0;的函数称为”纯虚函数”。

在定义一个表达抽象概念的基类时，有时无法给出某些函数的具体实现方法，就可以将这些函数声明为纯虚函数。

类内部定义了纯虚函数的类称为”抽象类”。

主要特点：抽象类只能作为基类来派生新类，不能声明抽象类的对象(既然都是一个抽象概念了，纯虚函数没有具体实现方法，故不能创造该类的实际的对象)。但是可以声明指向抽象类的指针变量或引用变量，通过指针或引用，就可以指向并访问派生类对象，进而访问派生类的成员(体现了多态性)。

作用：因为其特点，基类只是用来继承，可以作为一个接口，具体功能在派生类中实现。

函数重载与重写的区别：

函数重载是指函数名相同，但是函数的参数列表不同，在输入的参数不同时根据参数匹配调用相应的函数，称为函数重载。

重写指如果派生类中的方法，和基类继承来的某个方法的返回值，函数名，参数列表都相同，并且基类的方法是虚函数，那么派生类中的这个方法，自动处理为虚函数。这个操作称为”覆盖”，也称为”重写”。

覆盖是指在虚函数表中，虚函数地址的覆盖。在调用该方法时，会将虚函数表中基类的虚函数地址替换为派生类中重写的虚函数地址，所以调用的就是派生类中的函数。

虚析构函数

析构函数可以写为虚函数，因为在调用析构函数时，对象已经存在并且函数已经有地址了。而像构造函数，内联函数，静态成员函数是不可以写为虚函数的。

当基类的指针指向堆上的派生类对象时，应将析构函数写为虚析构函数。

如果不写为虚函数的话，通过delete想释放派生类对象的内存时，会导致派生类对象的析构函数无法调用，只调用了基类部分的析构函数，如果此时派生类的析构函数有释放额外系统资源的代码，那么就直接造成资源泄露了。但如果写为虚函数的话，在调用析构函数时，基类对象指针为派生类的虚函数指针，指向了派生类对象的虚函数表，发生动态绑定，调用了派生类的析构函数，成功释放了堆上的内存，而基类是定义在栈上的，函数调用完成后会自动调用基类的析构函数。

析构函数的调用顺序

先调用构造函数的后调用析构函数，后调用构造函数的先调用析构函数。

为何构造函数不能是虚函数

从存储空间角度看，虚函数地址需要存储在虚函数表中，而需要虚函数指针才能指向虚函数表，但是虚函数指针是存储在实例化的对象的内存中，而构造函数就是用于初始化一个对象的，所以在对象还没有实例化之前不存在虚函数指针，也就无法通过虚函数指针来调用构造函数了。

从使用角度看，虚函数主要用于对外的接口中，用基类对象指针或引用变量指向派生类对象，可以实现代码的复用，而在实例化一个对象时，构造函数是自动调用的，没有必要再去通过动态绑定调用构造函数了。

进程的虚拟地址空间

程序在经过编译链接完成之后会产生一个.out(windows下为.exe)的可执行文件，并且存放在磁盘上，而当要运行这个可执行文件时会把它加载到内存中，并且系统会产生一个进程。但是该文件不可能直接加载到物理内存中，而是由系统为它分配一部分虚拟内存，称为进程的虚拟地址空间。

供操作系统执行的二进制文件中，指令和数据(会生成符号存放在符号表中的)是分开存放的。原因如下：

  1. 程序执行之后，代码和数据可以被映射到不同属性的虚拟内存中。因为代码一般是只读的，而数据是可读可写的；
  2. 现代CPU有强大的缓存体系。程序和代码分离可以提高程序的局部性，增加缓存命中的概率；
  3. 还有最重要的一个原因是当有多个程序副本在运行的时候，只读部分可以只在内存中保留一份，这样大大节省了内存。

进程的虚拟地址空间大小共4GB，一部分是大小为1GB的内核空间，另一部分是大小为3GB的用户空间，注意用户空间是进程私有的，内核空间是共享的，即意味每个进程都有自己独立的用户空间，而内核空间只有一份并且所有进程共享。

堆与栈的区别：

栈：是由编译器在需要时自动分配，不需要时自动清除的变量存储区。通常存放局部变量、函数参数等。
堆：是由new分配的内存块，由程序员释放（编译器不管），一般一个new与一个delete对应，一个new[]与一个delete[]对应。如果程序员没有释放掉， 资源将由操作系统在程序结束后自动回收。

系统响应：对于堆，应知道系统有一个记录空闲内存地址的链表，当系统收到程序申请时，遍历该链表，寻找第一个空间大于申请空间的堆结点，删 除空闲结点链表中的该结点，并将该结点空间分配给程序（大多数系统会在这块内存空间首地址记录本次分配的大小，这样delete才能正确释放本内存 空间，另外系统会将多余的部分重新放入空闲链表中）。对于栈，只要栈的剩余空间大于所申请空间，系统为程序提供内存，否则报异常提示栈出。

空间大小： 堆是不连续的内存区域（因为系统是用链表来存储空闲内存地址，自然不是连续的），堆大小受限于计算机系统中有效的虚拟内存（32bit 系统理论上是4G），所以堆的空间比较灵活，比较大。栈是一块连续的内存区域，大小是操作系统预定好的，windows下栈大小是2M（也有是1M，在 编译时确定，VC中可设置）。

碎片问题：对于堆，频繁的new/delete会造成大量碎片，使程序效率降低。 对于栈，它是一个先进后出的队列，进出一一对应，不会产生碎片。

生长方向：堆向上，向高地址方向增长。栈向下，向低地址方向增长。

分配方式： 堆都是动态分配（没有静态分配的堆）。栈有静态分配和动态分配，静态分配由编译器完成（如局部变量分配），动态分配由alloca函数 分 配，但栈的动态分配的资源由编译器进行释放，无需程序员实现。

分配效率：堆由C/C++函数库提供，机制很复杂。所以堆的效率比栈低很多。栈是极其系统提供的数据结构，计算机在底层对栈提供支持，分配专门 寄存 器存放栈地址，栈操作有专门指令。

内存泄漏问题：

内存泄漏是指由于疏忽或错误造成了程序未能释放掉不再使用的内存的情况。内存泄漏并非指内存在物理上的消失，而是应用程序分配某段内存后，由于设计错误，失去了对该段内存的控制，因而造成了内存的浪费。

1、堆内存泄漏 （Heap leak）。对内存指的是程序运行中根据需要分配通过malloc, new等从堆中分配的一块内存，再是完成后必须通过调用对应的 free或者delete 删掉。如果程序的设计的错误导致这部分内存没有被释放，那么此后这块内存将不会被使用，就会产生Heap Leak。

2、系统资源泄露（Resource Leak）。主要指程序使用系统分配的资源比如SOCKET等没有使用相应的函数释放掉，导致系统资源的浪费，严重可导致系统效能降低，系统运行不稳定。

3、没有将基类的析构函数定义为虚函数。当基类指针指向子类对象时，如果基类的析构函数不是virtual，那么子类的析构函数将不会被调用，子类的资源没有正确是释放，因此造成内存泄露。

vector和list的区别

vector：

连续存储的容器，动态数组，在堆上分配空间

底层实现：数组

访问：支持随机访问

两倍容量增长：直接定义一个vector容器是不分配内存的，在push_back的过程中，由1扩容到2，2扩容到4，只要内存不够了就进行2倍扩容，但是扩容需要新建一个容器对象调用它的构造函数，并且要将原容器中的数据拷贝到新容器，还要调用原容器对象的析构函数，因为这些过程导致最终的效率是很低的。所以在使用vector时，需要先调用reserve()的方式进行内存预留，就省去了扩容所造成的时间浪费。并且迭代器也会失效。

对尾节点操作时时间复杂度低，为O(1)，但对其他节点操作时会涉及到拷贝问题，效率很低。

list：

动态链表，在堆上分配空间，每插入一个元数都会分配空间，每删除一个元素都会释放空间。

底层：双向链表

访问：不支持随机访问，只能快速访问头尾节点。

插入：很快，时间复杂度为为O(1)

删除：很快，时间复杂度为为O(1)

适用场景：经常插入删除大量数据

结构体内存对齐

第一个成员在结构体变量偏移量为0 的地址处。

其他成员变量要对齐到对齐数的整数倍：对齐数为Linux下为4与成员大小之间的较小值。

结构体大小一定是最大对齐数的整数倍。

内存对齐可以大大提升内存访问速度，是一种用空间换时间的方法。

内存不对齐会导致每次读取数据都会读取两次，使得内存读取速度减慢。

cpu把内存当成是一块一块的，块的大小可以是2,4,8,16 个字节，因此CPU在读取内存的时候是一块一块进行读取的，块的大小称为（memory granularity）内存读取粒度。

结构体和联合体的区别？

联合体是同个基本数据类型或复合数据类型公用同一片内存

特点：

  • 联合体的长度等于最长的成员的长度的整数倍；

  • 连续对联合体的不同成员赋值，将会覆盖已赋过值的成员。

结构体是存储在栈里面的吗？

结构体是一个自定义的数据变量，存储在哪里是依据在哪里定义的。若定义在全局区且初始化不为0，则存储在.data段，若未初始化或初始化为0，则存储在.bss段；若有static关键字修饰，则不论其定义在全局区或者函数体内，都是在.data段（初始化不为0）或者.bss段（未初始化或初始化为0）的。若是通过new来申请内存的话，则存储在堆区，否则在函数体内的话存储在栈区。

如何用联合体区分大段与小段？

大端模式：数据的高位，存放在地址的低位。（高位存低位，低位存高位）
小端模式：数据的高位，存放在地址的高位。（高位存高位，低位存低位）

在union中所有的数据成员共用一个空间，同一时间只能储存其中一个数据成员，所有的数据成员具有相同的起始地址。即上述的union虽然定义了两个成员，但其实这个union只占用了4个字节(32位机器中)，往a成员赋值，然后读取b就相当于读取a成员的低位第一个字节的值。如果机器使用大端模式，则u.a=1那a的最高字节值为1；如果机器使用小段模式，则u.a=1则a的最低位字节为1。上述可知b和a有相同的起始位，所以读取b如果等于1，则为小端模式，b为0则为大端模式。

智能指针

C++里面的四个智能指针: auto_ptr, shared_ptr, weak_ptr, unique_ptr 其中后三个是c++11支持，并且第一个已经被11弃用。

为什么要使用智能指针：

智能指针的作用是管理一个指针，因为存在以下这种情况：申请的空间在函数结束时忘记释放，造成内存泄漏。使用智能指针可以很大程度上的避免这个问题，因为智能指针就是一个类，当超出了类的作用域是，类会自动调用析构函数，析构函数会自动释放资源。所以智能指针的作用原理就是在函数结束时自动释放内存空间，不需要手动释放内存空间。

1. auto_ptr（c++98的方案，cpp11已经抛弃）

3.shared_ptr

shared_ptr实现共享式拥有概念。多个智能指针可以指向相同对象，该对象和其相关资源会在“最后一个引用被销毁”时候释放。从名字share就可以看出了资源可以被多个指针共享，它使用计数机制来表明资源被几个指针共享。可以通过成员函数use_count()来查看资源的所有者个数。除了可以通过new来构造，还可以通过传入auto_ptr, unique_ptr,weak_ptr来构造。当我们调用release()时，当前指针会释放资源所有权，计数减一。当计数等于0时，资源会被释放。

shared_ptr 是为了解决 auto_ptr 在对象所有权上的局限性(auto_ptr 是独占的), 在使用引用计数的机制上提供了可以共享所有权的智能指针。

成员函数：

use_count 返回引用计数的个数

unique 返回是否是独占所有权( use_count 为 1)

swap 交换两个 shared_ptr 对象(即交换所拥有的对象)

reset 放弃内部对象的所有权或拥有对象的变更, 会引起原有对象的引用计数的减少

get 返回内部对象(指针), 由于已经重载了()方法, 因此和直接使用对象是一样的.如 shared_ptr sp(new int(1)); sp 与 sp.get()是等价的

4.weak_ptr

weak_ptr 是一种不控制对象生命周期的智能指针, 它指向一个 shared_ptr 管理的对象. 进行该对象的内存管理的是那个强引用的 shared_ptr. weak_ptr只是提供了对管理对象的一个访问手段。weak_ptr 设计的目的是为配合 shared_ptr 而引入的一种智能指针来协助 shared_ptr 工作, 它只可以从一个 shared_ptr 或另一个 weak_ptr 对象构造, 它的构造和析构不会引起引用记数的增加或减少。weak_ptr是用来解决shared_ptr相互引用时的死锁问题,如果说两个shared_ptr相互引用,那么这两个指针的引用计数永远不可能下降为0,资源永远不会释放。它是对对象的一种弱引用，不会增加对象的引用计数，和shared_ptr之间可以相互转化，shared_ptr可以直接赋值给它，它可以通过调用lock函数来获得shared_ptr。
